Overview

VARIANT compares a set of viral genome sequences to a curated reference genome and proteome to identify and classify mutations at both the nucleotide and amino acid level. It also optionally predicts Programmed Ribosomal Frameshifting (PRF) sites.

Three analysis steps are performed internally:

  1. SNP detection — each genome in the MSA is compared position-by-position to the reference; substitutions, insertions and deletions are recorded.
  2. Protein-level translation — identified SNPs are mapped onto the annotated ORFs from the proteome file and amino acid changes are inferred.
  3. Hot / row mutation classification — clusters of adjacent mutations (row) and frequently recurring mutations across many genomes (hot) are flagged.

An optional fourth step runs the PRF scanner to report candidate slippery sequences in each reference segment.

Input file specifications

Reference genome .fasta / .fa / .fna
  • Standard FASTA format with one or more sequences (one per segment for multi-segment viruses).
  • Sequences must contain only IUPAC nucleotide characters: A C G T U R Y S W K M B D H V N and gap -. Any other character causes the upload to be rejected.
  • The header line (>identifier) is used as the reference ID throughout the analysis.
Proteome .fasta / .fa / .faa
  • Multi-FASTA file where each entry is one protein (ORF) encoded in the reference genome.
  • Sequences must contain only standard one-letter amino acid codes: A C D E F G H I K L M N P Q R S T V W Y X * and gap -. Any other character is rejected.
  • Header lines must be consistent with the ORF annotations used internally.
MSA file .txt / .fasta / .fa / .aln / .clustal
  • A pre-computed multiple sequence alignment of the genomes you wish to compare.
  • Accepted formats: ClustalW (preferred) or multi-FASTA. The reference sequence must be included.
  • All sequences must be the same length (padded with - where necessary). Tools such as MAFFT, MUSCLE or ClustalW can generate these alignments.
  • Genome IDs in the MSA header lines are extracted automatically and offered as autocomplete suggestions in the analysis form.
File validation: VARIANT validates the biological content of every upload. Files containing invalid characters are rejected immediately with a description of the offending characters.

Built-in viruses vs Custom viruses

Built-in viruses

SARS-CoV-2, HIV-1, Chikungunya, ZaireEbola and H3N2 ship with curated reference genomes and proteomes. These cannot be replaced through the upload form — doing so would corrupt the shared reference for all users.

You can upload additional MSA files for built-in viruses to run comparative analyses on your own sequence cohorts.

Custom viruses

Use the Custom Virus panel (Data Upload tab) to register any virus not in the built-in list. You must supply all three files: reference genome, proteome and MSA.

Custom viruses support single-segment and multi-segment configurations. Name them clearly — the name is used as a directory label and in all output filenames.

Example use cases: human mtDNA, bacteriophages, novel emerging pathogens, custom SARS-CoV-2 subsets.

Output files

All output is written to result/{virus_name}/ and can be downloaded as a ZIP from the History tab.

Visualizations

After a completed analysis, open the Analysis & Visualization tab, select the virus (and optionally a genome ID) in the Generate Visualization panel, then click the button. Three interactive Plotly HTML plots are displayed:

  • Mutation Analysis — genome-wide overview with per-protein mutation bars. Hover over each bar for position-level detail.
  • Row / Hot Mutations — highlights mutation clusters (row) and high-frequency sites (hot) across the proteome.
  • PRF Regions — shows predicted frameshifting sites along the genome with confidence scores.

All three plots are interactive: zoom, pan, hover tooltips, and download as PNG via the Plotly toolbar. Switching to a different virus re-generates all three plots fresh.

RNA Dual Graph tab

The RNA Dual Graph tab assigns a topology ID to an RNA secondary structure described in dot-bracket notation.

Input format (dot-bracket)
>MY_RNA
AUGCAUGCAUGCAUGC
((((....))))....
  • Line 1: sequence name (preceded by >).
  • Line 2: nucleotide sequence (A, U, G, C).
  • Line 3: dot-bracket structure. Supported brackets: ( ), [ ], { }.
  • Use & to separate strands in a multi-strand structure.

The pipeline runs: dot-bracket → DSSR format → CT file → dual graph library match by Laplacian eigenvalue + graph isomorphism. Matched topology IDs are shown in a table; click Plot to display the dual graph topology.

Frequently asked questions

The button is enabled only when all three required files (reference genome, proteome, MSA) have been uploaded for the selected virus. Check the upload checklist on the Data Upload tab.

Built-in viruses use their own curated reference genome and proteome regardless of what MSA you upload. If you want to analyse a different organism, create a Custom Virus and upload all three files. Uploading a reference genome or proteome for a built-in virus is now blocked by the server with an explanatory error message.

VARIANT validates file content: reference genome files must contain only IUPAC nucleotide characters and proteome files must contain only standard amino acid characters. Remove or correct the flagged characters before re-uploading. Common causes: protein sequences uploaded as reference genome (wrong slot), quality scores or metadata mixed into sequence lines, or Windows-style whitespace.

Error messages are shown in the History tab next to the failed job. Common errors: "No MSA file configured" means the upload checklist is incomplete; "Reference genome file is missing" means the custom virus config has no reference genome set (re-upload the file). Contact the server administrator if errors persist.

Each time you click Generate All Visualizations the plots are regenerated with a fresh timestamp so the browser does not cache old content. If you still see stale content, perform a hard refresh (Ctrl+Shift+R / Cmd+Shift+R).

Job records are now persisted to disk and survive server restarts. If a download still fails, the result files may have been deleted from the server (result directories are shared and overwritten by subsequent analyses of the same virus). Re-run the analysis and download immediately after completion.

  1. Click Create Custom Virus in the Data Upload tab and choose single-segment.
  2. Upload your reference genome (the mitochondrial genome FASTA).
  3. Upload your proteome (all mitochondrially encoded protein sequences in one FASTA).
  4. Upload your MSA (ClustalW alignment of the genomes you want to compare).
  5. Switch to the Analysis tab, select your custom virus, and run the analysis.
